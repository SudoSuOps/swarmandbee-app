Cookies

Last updated: 2026-05-17

The short version

We use no third-party cookies. We run no analytics. We run no ad tracking. There is no GDPR-style consent banner because there is nothing to consent to.

What about first-party state?

The only state our site sets is a session cookie if you log into the admin dashboard at orderbook.swarmandbee.ai — that page is protected by HTTP Basic Auth, and the browser maintains a session cookie for that auth context. That cookie is scoped to orderbook.swarmandbee.ai, contains no tracking identifier, and exists only to keep you logged into the admin tool while you use it.

That's it

The main swarmandbee.ai site, the bakery surface at bakery.swarmandbee.ai, the bounty intake at bounty.swarmandbee.ai, and the identity page at identity.swarmandbee.ai — none of them set cookies. They are static surfaces served behind a Cloudflare edge. Cloudflare may set its own infrastructure cookies (e.g. __cf_bm) as part of bot-management; those are Cloudflare's, not ours.

Why this matters

The same discipline that produces a defendable training corpus — no round-number lies, every claim has a receipt — applies to how the site itself is built. We sell sovereign data infrastructure. It would be a tell if our own surface leaked behavior to ad networks.
